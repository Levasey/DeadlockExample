# Deadlock Example with Try-Lock Solution

Демонстрационный проект, иллюстрирующий проблему взаимной блокировки (deadlock) в многопоточном программировании на Java и её решение с использованием метода `tryLock()` из пакета `java.util.concurrent.locks`.

## 📋 О проекте

Программа моделирует классическую ситуацию deadlock при параллельных переводах средств между двумя банковскими счетами. Два потока одновременно выполняют транзакции, блокируя счета перед переводом. Без надлежащей синхронизации это приводит к взаимной блокировке.

### 🏗️ Компоненты системы

- **`Account`** — класс банковского счета с методами депозита и снятия средств
- **`Runner`** — основной управляющий класс, реализующий логику потоков и блокировок
- **`Main`** — точка входа в приложение

## 🔍 Проблема Deadlock

В классической реализации (без использования `tryLock()`) возникает deadlock:

```
Поток 1: захватывает lock1 → ожидает lock2
Поток 2: захватывает lock2 → ожидает lock1
```

Оба потока блокируют друг друга, создавая ситуацию вечного ожидания.

## 💡 Решение с tryLock()

В проекте реализован механизм безопасного захвата блокировок с повторными попытками:

```java
private void takeLock(Lock lock1, Lock lock2) {
    boolean firstLockTaken = false;
    boolean secondLockTaken = false;
    
    while (true) {
        try {
            firstLockTaken = lock1.tryLock();
            secondLockTaken = lock2.tryLock();
        } finally {
            if (firstLockTaken && secondLockTaken) {
                return;
            }
            if (firstLockTaken) {
                lock1.unlock();
            }
            if (secondLockTaken) {
                lock2.unlock();
            }
        }
        
        try {
            Thread.sleep(1);
        } catch (InterruptedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

### ✅ Преимущества подхода

- **Неблокирующая проверка** — `tryLock()` немедленно возвращает управление, если блокировка недоступна
- **Автоматическое освобождение** — частично захваченные блокировки всегда освобождаются
- **Предотвращение deadlock** — потоки никогда не удерживают одну блокировку в ожидании другой
- **Устойчивость к состоянию гонки** — повторные попытки гарантируют выполнение операций
